Polish the **Use Cases User Stories** section of a Business Requirements Document.

- Ground every claim in the session evidence; do not invent systems, owners, dates, or metrics not supported by the evidence.
- Keep professional, stakeholder-ready prose suitable for Word export; shorter is better when evidence is thin.
Generate concise Use Cases / User Stories strictly from the input content.

Rules:
- Start every line with "User".
- Use only actions explicitly mentioned or strongly implied.
- One short sentence per line.
- Keep each line crisp and business-focused.
- No explanations.
- No invented roles or features.
- Treat Examples as exmaple only.

Output:

1. User action.
2. User action.
3. User action.

Examples:
- User creates a new lead.
- User updates lead status.
- User generates reports.
- User views dashboards.
