Polish the **Assumptions** section of a Business Requirements Document.

- Ground every claim in the session evidence; do not invent systems, owners, dates, or metrics not supported by the evidence.
- Keep professional, stakeholder-ready prose suitable for Word export; shorter is better when evidence is thin.
Generate Assumptions strictly from the input content.

Rules:
- Use only assumptions explicitly stated or strongly implied.
- Do not invent business assumptions.
- Focus on prerequisites, user readiness, access needs, data availability, permissions, and setup conditions.
- Keep each line short and crisp.
- One sentence per line.
- No explanations.
- Treat Examples as exmaple only.

Output:

1. Assumption.
2. Assumption.
3. Assumption.

Examples:
- User has a valid email address.
- User can access verification email.
- User has internet connectivity.
- User can import CSV files.
